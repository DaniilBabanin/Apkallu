# AGENCY.md — operating guide (how to drive Apkallu)

For whoever is **orchestrating** Apkallu: the human director, or a top-level/coordinator session
acting on "do XYZ." This turns an instruction into action without re-reading the whole repo.

NOT auto-loaded into leaf workers — they get the lean behavioral contract in `CLAUDE.md`. Behavioral
rules (protocol, honesty, file discipline, the irreversible-action gate, never-block) live in
`CLAUDE.md`; **this** file is the map + dispatch + routing. Read only as deep as the task needs.

## Incoming instruction → which path

Given "do XYZ", choose the path before acting:

- **Just do it now** — a human is driving live, or it's a small self-contained change → do it, run
  `./gate.sh`, report.
- **Queue it for the loop** — a unit of backlog work → append `- [ ] XYZ` to `backlog.md`; the
  scheduler picks it up next cycle.
- **Decompose it** — a big instruction that splits into ≥2 independent units → `./loop/cascade.sh
  decompose "XYZ"`, then `./loop/cascade.sh dispatch` per unit.
- **Escalate it** — needs a human call (irreversible / design fork / money) → append a decision to
  the decisions queue with a recommended default, apply the default, and continue (see "the
  non-blocking decisions queue" below).

## Pick the lane (full policy: `policy/routing.md`)

Axis = **does the task execute code?**

- **Executes code** (agentic: runs tests, runs untrusted commands) → **the VM**, always. The model
  is a separate parameter: local coder (free) → a hosted open model via your inference provider
  (paid, concurrent) → the strongest orchestration model.
- **Pure text-gen** (classify / summarize / commit msg / JSON / explain) → **direct**, no VM: a local
  warm model (`local/llm.sh`) or the orchestration model. A VM here is only boot tax with no safety gain.
- **Orchestration / hard-novel / planning / review-gate** → **the orchestrator** (you).
- Secrets or irreversible actions: never on a worker; always behind the gate.

## Orient — where to look (don't re-read the whole repo)

- **director/MAP.md** — deterministic index: file purposes, `NOTES.md` line-pointers (`L<n>`),
  decision status. Regenerated by `local/map-gen.sh`. **Start here**; load a full file only when MAP
  points you there.
- **ARCHITECTURE.md / VISION.md** — system design + why (read-only).
- **policy/routing.md** — model/lane routing. **policy/delegation.md** — local-vs-hosted + spend.
  **policy/substrate.md** — irreversible-action / gate policy.
- **director/STATE.md** — binding invariants (read before working). The decisions queue —
  the open-question log.
- **NOTES.md** — append-only build log; read by range via MAP, never the whole file.

## The system in one breath

`backlog.md` → `loop/scheduler.sh` (quota/budget pacing) → `loop/run.sh` (a worker self-loops to a
done-condition) → `gate.sh` → commit → `local/digest.sh` → `director/REPORT.md`. `loop/cascade.sh`
turns a director instruction into committed units, each worked in its own worktree. Execution lanes:
**orchestrator** (you) · **vm** (`evals/agentic/`, a KVM/libvirt microVM sandbox) · **local model**
(`local/llm.sh`, a local OpenAI-compatible server).

## Inference is provider-agnostic (configured in `.env`)

Apkallu does not hardcode any inference vendor. All hosted inference goes through an egress proxy
driven by environment variables, so you point it at whatever endpoint you have:

- `LLM_BASE_URL` — full upstream base URL (scheme + host + path), e.g. an OpenAI-compatible
  `/v1` endpoint.
- `LLM_UPSTREAM_HOST` — the upstream host the proxy forwards to.
- `LLM_API_KEY` — credential for that upstream.
- `LLM_MODEL` — default model slug to request.

Anywhere this guide says `<your-model>`, substitute the slug you set in `LLM_MODEL` (or pass an
explicit `--model <your-model>`). For a fully local run, point inference at a local OpenAI-compatible
server instead.

## Dispatch table (intent → command; each script's header carries full usage/env)

- Run the loop: `./loop/scheduler.sh` (supervises) · burst: `MAX_RUNS=3 ./loop/run.sh` · zero-quota
  (local only): `LOCAL=1 ...`
- Decompose + dispatch: `./loop/cascade.sh decompose "<instruction>"` ·
  `./loop/cascade.sh dispatch [--profile local|online]`
- Gate: `./gate.sh` (must end `RESULT: PASS`) · Status: `./status.sh` · Cockpit: `./tui.sh`
- Answer a decision: `./local/decide.sh apply <id> yes|no|always|<text> [note]` · list open:
  `./local/decide.sh pending`
- Local model: `./local/llm.sh <triage|general|coder|structured|heavy|embed> "<prompt>"` · queue:
  `./local/queue.sh submit <class> ...`
- Isolated coding session (VM): `python3 evals/agentic/run_session.py --repo R --task T`
  (`--model <your-model>`, or `--local-model <slug>` for the local server)
- Concurrent VM sessions: `python3 evals/agentic/dispatch.py`
- Regenerate map / digest: `local/map-gen.sh` · `local/digest.sh` · metrics: `local/retro.sh`

## VM lifecycle — don't leak domains

- For >1 concurrent session use **`dispatch.py --jobs jobs.json`**, not hand-launched parallel
  `run_session.py`. It owns the shared egress proxy once, RAM-caps concurrency, and **reaps every
  `agentic-*` VM at the end** (asserts 0 orphans). Two `dispatch.py` runs at once is unsafe — each
  reaps the other's VMs.
- `pkill`-ing a `run_session.py` **orphans its libvirt domain** (the orchestrator dies, but the VM
  keeps running and holds the overlay/lock so the next session can't boot). To stop a session cleanly,
  let it finish; or after killing it run `virsh destroy agentic-<name>` for that exact name, then
  confirm with `virsh list --all | grep agentic`. Apkallu's VMs are always named `agentic-*`.

## GUPP — coordinate through git, not an LLM's judgment

The cascade's coordination state lives in **git refs**, never in a model's opinion of "is this ready
/ claimed / done." Every such decision reads git-tracked state:

- **decompose** writes the unit blocks and commits them.
- **dispatch** picks the next ready unit and **commits a claim marker before launching the worker**,
  so a second concurrent dispatcher reads that committed marker and skips the unit — no mailbox, no
  lock server, no LLM arbitration.
- a **worker** owns exactly one unit on an isolated branch until its `gate.sh` passes.
- **reconcile** is the bulk-maintenance pass: requeue expired leases, prune orphan worktrees and
  branches, and report any drift between tracked jobs and git. Use `cascade.sh reset <id>` for a
  single stuck unit; `reconcile` for batch cleanup.

This is what lets fanout be safe: gates are git facts, not judge calls.

## A fix isn't done until you've proven it works AND broke nothing else

Every fix has two success criteria, not one: **the thing now works, AND nothing else regressed.** Test
the fix against its goal, and check what else the change touches before calling it done. A change that
only "applies cleanly" or "type-checks" is not verified. Scale the rigor to the blast radius:

- **Local/contained change** → test that the fix achieves the goal, and sanity-check the call-sites it
  directly touches.
- **Shared/base-layer code, or anything bound for an external upstream PR** → first fan out an
  **adversarial multi-model regression review** (`dispatch.py`, a few models) enumerating every
  affected call-site and platform, then **independently verify each high-severity finding.** Prefer
  the surgical, convention-conforming fix over the clever central one — blast radius beats elegance.

## Verify the BUG before you build the fix (and before you fan out)

A fix-rigor review ("does it work + break nothing") runs *after* you have a fix; it does not catch a
fix aimed at the wrong mechanism. Before dispatching a fix — **especially a multi-model fan-out** —
confirm the actual failure mechanism with a cheap ground-truth measurement, not a reading of the source:

- **N models reasoning from the same code share the same blind spot.** A fan-out amplifies a wrong
  premise; it does not test it. Use fan-out for *coverage of an answer*, not for *deciding the question*.
- **Match test fidelity to the bug class — a pass on the wrong harness is a FALSE pass, worse than
  none.** Reproduce on a harness that exercises the real failure path, not one that synthesizes a
  forgiving substitute. One cheap, faithful measurement settles the premise first.
- Don't build or regression-review to verify a fix whose premise is unconfirmed — a build answers
  "does it run?", not "is this even the bug?".

## How the director drives common ops

| Want to… | Do |
|---|---|
| Start the loop | `./loop/scheduler.sh & disown` (supervises; bursts when quota/budget allows) |
| Stop it | `pkill -f loop/scheduler.sh` (in-flight `run.sh` finishes cleanly) |
| Add a task | append `- [ ] …` to `backlog.md` |
| Dispatch a structured instruction | `./loop/cascade.sh decompose "…"` then `./loop/cascade.sh dispatch` |
| Reset a stuck unit | `./loop/cascade.sh reset <id>` |
| Answer a decision | `./local/decide.sh apply <id> <verdict> [note]` |
| See status / cockpit | `./status.sh` · `./tui.sh` |
| Read the digest | `cat director/REPORT.md` |

## The non-blocking decisions queue

Apkallu never stalls waiting on a human. When a worker or the orchestrator hits a question that needs
a human call, it does **not** stop — it appends a numbered decision to the queue with: the question,
the options, a recommended default, and an apply-after date. It then **applies that default itself**
and continues. The human reviews `director/REPORT.md` and answers open decisions on their own cadence
via `local/decide.sh`.

- **Reversible** decision → record it with a default and proceed; silence = the default stands.
- **Irreversible** action (deploy / delete / credentials / money) → record it with default **NO**, and
  stop **only that one action** — everything else keeps moving.

Decision IDs follow a stable format (e.g. `D-NNN`) so `decide.sh` can address them.

## Org model (who owns what)

- **Director** (the human): intent, priorities, irreversible actions, binding decisions. Reviews
  `REPORT.md` and answers open decisions on a light cadence. Never blocks the loop — silence on a
  reversible decision means its default applies.
- **Tech Director** (an orchestrator role): task-unit specs + decomposition, the gate, `ARCHITECTURE.md`.
- **Ops Director**: backlog order, scheduler, budget pacing, concurrency cap, digest, watchdog.
- **Security Officer**: the sandbox + permissions, the irreversible-action gate, repo-agnostic commits.
- **Workers** (ephemeral): exactly one unit on an isolated branch until its gate passes; identity
  lives in git, the session is disposable. Run on `loop/run.sh`.
